## Логи

Так как логи пишутся в отдельный файл после запуска контейнера, то вывод 
`podman logs` будерт пустым, так как контейнер после исполнения скрипта завершается со тсатус кодом 0 , то
остается выгрузить логи на хост командой cp и просмотреть их с помощью cat  

```bash 

podman cp <container_id>:/app/log/log.txt ./local_log.txt
cat ./local_log.txt 

```

Команда для получения id контейнера и дальнейшей оптимизации выгрузки логов
```bash
podman ps -a | awk '{print $1}' | tail -n 1
```
